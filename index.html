What is the Independent Set Problem?

Given an undirected graph G = (V, E) and an integer k, does there exist a subset S ⊆ V with |S| ≥ k such that no two vertices in S share an edge?

IndependentSet(G, k) = ∃ S ⊆ V : |S| ≥ k ∧ ∀ u,v ∈ S : (u,v) ∉ E

In other words, find the largest group of vertices where nobody in the group is directly connected to anyone else in the group. Each vertex is independent — no edges among the chosen set.

🧩 Intuitive Analogy

Imagine a party where some pairs of people are enemies (edges). You want to invite the maximum number of guests so no two guests hate each other.

The Independent Set is the guest list. No two invited guests can be connected by a conflict edge.

⚙️ Real-World Applications

📡 Wireless Channel Assignment 🖥️ Register Allocation 🧬 Protein Compatibility 👥 Social Network Analysis 📦 Scheduling (conflict-free) 🎮 Game Theory (stable strategies)

Example: Two antennas on the same channel interfere (edge). An independent set = a set of non-interfering antennas all sharing one channel.

🔗 Connection to Vertex Cover

These are complementary problems. If S is an independent set, then V \ S is a vertex cover, and vice versa.

S is an IS ⟺ V \ S is a Vertex Cover

Vertex Cover: every edge touches at least one covered vertex.

Independent Set: no edge has both endpoints selected.

This complement relationship is the key to the NP-completeness reduction.

🏔️ Why Is It Hard?

The naive brute-force approach checks all 2n subsets. With 50 vertices that's more than 1015 checks. With 100 vertices: 1030.

No polynomial-time exact algorithm is known. Independent Set is NP-complete — among the hardest problems in NP.

We use a greedy heuristic in practice, which is fast but may miss the optimal solution.

NP-Completeness Proof

What Does NP-Complete Mean?

A problem is NP-complete if: (1) it is in NP — a "yes" answer can be verified in polynomial time, and (2) it is NP-Hard — every problem in NP reduces to it in polynomial time. We prove (2) by reducing from one known NP-complete problem: Vertex Cover.

Step 1: Independent Set ∈ NP
Membership Proof

To show IS ∈ NP, we describe a certificate and a polynomial-time verifier.

Certificate: A subset S ⊆ V (a list of vertex indices).

Verifier (runs in O(n²)):

function Verify(G, k, S):
// Check 1: Is |S| ≥ k?
if |S| < k: return false // O(1)
// Check 2: No two vertices in S are adjacent
for each pair (u, v) in S × S, u ≠ v: // O(k²) ≤ O(n²)
if (u, v) ∈ E: return false // edge found → NOT independent
return true // valid IS of size ≥ k ✓

Why polynomial time? The inner loop runs at most k² ≤ n² iterations. Edge lookup is O(1) with an adjacency matrix. Total: O(n²) — polynomial in input size.

✅ Conclusion: Independent Set ∈ NP.

Step 2: NP-Hardness via Reduction from Vertex Cover
Reduction Proof

We show Vertex Cover ≤p Independent Set. Since Vertex Cover is NP-complete, this makes IS NP-Hard.

3
Step 3: Correctness — Forward Direction (⇒)
VC → IS

Claim: If G has a vertex cover C with |C| = k, then G has an independent set S = V \ C with |S| = |V| − k.

  • Let C be a vertex cover with |C| = k.
  • Define S = V \ C. Then |S| = |V| − k. ✓
  • Why is S an independent set? Take any two vertices u, v ∈ S = V \ C. Since C is a vertex cover, every edge has at least one endpoint in C. But u ∉ C and v ∉ C, so the edge (u, v) cannot exist in E (it would require at least one of them to be in C). ✓

Visual Example: In a 5-vertex path A–B–C–D–E, the VC {B,D} of size 2 covers all edges. The complement {A,C,E} is an IS of size 3 = |V|−k = 5−2.

4
Step 4: Correctness — Backward Direction (⇐)
IS → VC

Claim: If G has an independent set S with |S| = |V| − k, then G has a vertex cover C = V \ S with |C| = k.

  • Let S be an IS with |S| = |V| − k.
  • Define C = V \ S. Then |C| = |V| − |S| = |V| − (|V| − k) = k. ✓
  • Why is C a vertex cover? Take any edge (u, v) ∈ E. Since S is an independent set, u and v cannot both be in S. Therefore, at least one of u, v must be in V \ S = C. Every edge is covered. ✓

Biconditional Summary:

G has a vertex cover of size k ⟺ G has an independent set of size |V| − k
✓
Conclusion: Independent Set is NP-Complete

Algorithm Comparison

Greedy vs Brute Force

Brute force finds the exact maximum independent set but takes exponential time O(2n · n²). Greedy uses the minimum-degree heuristic in polynomial time O(n²+m), but may miss the optimal. The comparison tab shows both pseudocodes, runs both algorithms on the same graph, and highlights when greedy fails.

🔴 Brute Force Exact Algorithm (CLO-4)

🧠 Algorithm Design — Greedy Paradigm (CLO-3)

Intuition: When you pick vertex v for the IS, all its neighbors become ineligible. Choosing the vertex with fewer neighbors first eliminates fewer candidates, leaving more options in future steps.

Time Complexity: Each iteration finds min-degree vertex and removes neighbors in O(n + m). At most n iterations → O(n² + nm).

Approximation Guarantee: Greedy finds IS of size ≥ n/(Δ+1), where Δ is maximum degree. Not a constant-factor approximation in general.

Does it use recursion? No. The greedy approach is iterative — a while loop that terminates in at most n iterations, each time shrinking the active subgraph. Recursion is not needed here.

Brute force recursion? Brute force can be written recursively (enumerate subsets via include/exclude recursion), but the iterative bitmask implementation shown here is equivalent and more cache-friendly.
